# 前端设计与交互审计

审计范围：`frontend/src`。基线扫描发现 105 处 `transition-all`、26 处 hover 位移、26 处内联入场动画、173 处 9–10px 文本、9 处 `line-clamp`。以下按可用性影响排序，修改以 `DESIGN.md` 为准。

## P0：可用性、阅读与响应式

| 优先级 | 页面或组件 | 当前问题 | 修改方案 | 设计依据 |
| --- | --- | --- | --- | --- |
| P0 | `TranscriptClaimsPanel` | 桌面观点卡依靠首次 hover/focus 触发展开，使用绝对定位和固定 400px 高度；1024–1280px 易裁切，键盘用户进入前难以预判结构 | 改为响应式网格，卡片自然流布局；主要内容和收藏/隐藏/详情始终可见 | 内容不依赖 hover；1024px 不使用绝对定位展开 |
| P0 | `TranscriptClaimsPanel` / `ClaimSpreadCard` | 固定 `h-[368px] w-[17.5rem]` 与多层 line-clamp 截断观点、理由和标签 | 改为最小高度、自适应宽度；保留合理摘要但不裁切主要观点 | 观点本身优先于卡片整齐度 |
| P0 | `TranscriptWorkspace` | 所有摘要、资料、观点、Speaker、逐字稿均为同重量白卡，长页缺少任务导航；对象标题在桌面 `truncate` | 增加页内锚点/分区导航，开放标题换行，降低资料与 Speaker 权重，稳定阅读顺序 | 长页面保留上下文；标题不可隐藏关键信息 |
| P0 | `AudioPlaybackBar` | 波形播放时逐条持续动画，没有额外信息价值；透明 range 的 focus 不可见，紧凑版时间为 9px | 移除持续波形脉动；为 seek 控件提供可见 focus/轨道状态；时间最小 11px | 动画服务信息；键盘 seek 可见 |
| P0 | `ModalShell` | Dialog 只有淡入，没有区分 backdrop/panel easing；全局 reduced-motion 将所有过渡压到 0.01ms，连有意义颜色反馈也移除 | 拆分 backdrop/panel 动效；reduced-motion 仅移除位移/缩放/持续动画，保留短颜色与透明度 | Emil 动效与无障碍原则 |
| P0 | 全局页面框架 | Sidebar + 固定视口高度下，窄屏主内容可用宽度有限；多个组件包含固定宽度、`overflow-hidden` | 主区域统一 `min-w-0`，窄屏检查固定宽度，允许控件换行 | 无横向意外滚动 |

## P1：明显的视觉与交互不一致

| 优先级 | 页面或组件 | 当前问题 | 修改方案 | 设计依据 |
| --- | --- | --- | --- | --- |
| P1 | `index.css` 与全前端 | 39 个文件使用 `transition-all`，不同属性和时长混用 | 建立 120/180/240ms token；只过渡颜色、边框、透明度、transform 或 width | 精确、可打断、性能稳定 |
| P1 | `Header`、各页面卡片 | Header 每次路由切换入场；20 个文件存在页面/列表入场动画 | Header 和高频页面区块静态呈现；仅保留 Modal、短状态与骨架动画 | 高频操作不做入场动画 |
| P1 | 全局按钮 | 26 处 hover 上浮，有些按钮无 active 反馈，圆角与高度不一致 | 去掉 hover 位移，统一短颜色过渡与 `active:translate-y-px`；按主/次/危险分类 | 即时按压反馈；不让所有卡片浮起 |
| P1 | 全局微文案 | 173 处 9–10px 文本，其中部分承载状态、筛选和字段 label | 关键标签提升到 11px；10px 仅留装饰编号；提高低对比度辅助文字 | 长时间阅读与 WCAG AA |
| P1 | `TranscriptSummaryPanel` | 摘要、章节、人物观点、重点均为完整白卡，重复标题结构；页签为 10px | 保留单列阅读宽度，减少内部卡片实度，提升控制字号，进度只过渡 width | Notion 阅读宽度 + Linear 控件精度 |
| P1 | `PodcastMetadataEditor` | 展示态占一整张高权重卡，字段层级弱；编辑按钮与保存反馈不统一 | 改为较轻的上下文区/定义列表；表单 label 11px，保存状态可读 | 元数据不与正文争夺权重 |
| P1 | `SourceCollection` | 新建入口和内容库按钮均 hover 浮起；QuickGenerate 同时在页面和全屏 Modal 重复出现，首屏任务焦点分散 | 移除浮起，强化“从哪里开始”和当前任务；页面内保持简要入口，全屏承载完整采集 | 工作台首屏只表达起点与当前任务 |
| P1 | `ContentLibrary` / 历史列表 | 分类导航卡片较重，记录仍以多张白卡呈现，文件名/状态/时间/下一步的层级不够稳定 | 分类控件收紧为生产工具式 segmented nav；记录用轻行项目和常驻操作 | 大量内容扫描优先 |
| P1 | `VoicePresets` / 音色卡 | 多套卡片与 tab 尺寸，描述 line-clamp，试听/选择/管理表达不统一 | 统一资产标题、类型 pill、描述、试听和选中态；小屏单列 | ElevenLabs 式音频资产层级，但保留本项目视觉 |
| P1 | 长任务组件 | 总结、观点分析、生成、排队的状态组件各自实现 | 抽取轻量 `TaskProgress` / `StatusPill`，统一阶段、进度、错误语义 | 系统状态跨流程一致 |

## P2：细节优化

| 优先级 | 页面或组件 | 当前问题 | 修改方案 | 设计依据 |
| --- | --- | --- | --- | --- |
| P2 | `Sidebar` | 导航在桌面字号偏大、active 阴影较重，图标以抽象色点表达 | 收紧字号/间距，active 使用边框与浅底；保留现有结构 | Linear 式高密度导航 |
| P2 | 空状态 | 多处只有“暂无数据”，辅助文字对比度低且缺少下一步 | 统一 `EmptyState`，支持原因、下一步和 action | 空状态应可行动 |
| P2 | Section Header | 色点、标题字号和 margin 重复且略有差异 | 抽取 `SectionHeading`，只覆盖稳定重复结构 | 小而明确的共享组件 |
| P2 | 状态 Pill | 9px、padding、颜色透明度分散 | 抽取 `StatusPill`，最小 11px，颜色 + 文本双重表达 | 一致状态语言 |
| P2 | 卡片阴影 | 列表项、主要区块、导航选中态多处共用 `shadow-card` | 阴影仅留主要区块；行项目和选中态优先边框/背景 | 减少白卡堆叠和视觉噪声 |
| P2 | 错误动画 | 多处 `animate-shake` 可能在重渲染时重复触发 | 仅错误首次出现时触发，reduced-motion 移除 transform | 错误反馈明确但不打扰 |
| P2 | 测试覆盖 | 样式与响应式规则多靠人工，观点卡已有测试但依赖旧展开结构 | 更新观点、Modal、播放器测试；增加语义状态与键盘焦点断言 | 关键交互可回归 |

## 实施批次

1. 全局 token、focus、reduced-motion、Header、Sidebar、Modal 与共享 UI。
2. Transcript Workspace、摘要、逐字稿、观点、详情与元数据。
3. 统一播放器与音色资产表达，移除无意义波形动画。
4. 工作台和内容库的信息层级、空状态与常驻下一步。

## 2026-07-18 实施与对抗验收

本轮在 `codex/content-workbench-reading-experience` 分支继续完成“内容创作工作台”方向，重点不是换皮，而是重建用户读取信息和继续行动的顺序。

### 已落地

- 工作台改为“收集素材 → 提炼与写作 → 成稿或音频”，移除重复的第二份 `QuickGenerate`，入口滚动并聚焦唯一采集控件。
- 内容库按“成稿与音频 / 转录与整理 / 观点与项目”组织，列表优先标题、状态、时间与下一步；移动端记录改为自然堆叠。
- 长内容工作区使用“标题与状态 → 核心摘要 → 结构化导读 → 资料 → 观点 → 说话人 → 逐字稿”的阅读顺序；摘要、观点证据和逐字稿采用 15px / 1.9 行高与 48rem 阅读上限。
- 建立 `ui-page-title`、`ui-section-title`、`ui-body`、`ui-reading-body`、`ui-metadata`、`ui-control-label` 六档语义排版角色，并同步到 `DESIGN.md` 与前端 skill。
- GitHub 外链具备明确容器、40px 点击目标、键盘焦点、短箭头位移与按压反馈；reduced-motion 下移除 transform。
- 播放进度 range 保留原生键盘操作，并让可视轨道在 focus-visible 时出现明确焦点环。
- 动效统一到 120/180/240ms；移除列表 stagger、`transition-all`、历史宽泛属性过渡与 hover 浮起。

### 对抗验收结果

- `390×844`、`1024×768`、`1440×900` 的工作台与真实播客长文均无页面级横向溢出；移动端分区导航只在自身容器内横向滚动。
- 使用真实的 129 轮、22 条观点播客检查长标题、摘要、章节、观点和元数据；标题自然换行，分区跳转后目标位于 sticky 导航下方。
- GitHub 外链键盘聚焦时为 40px 高、2px 可见 outline；箭头完成约 3px 的右上位移。
- 验收页面没有 console warning/error；可见文字计算字号未发现小于 11px 的元素。
- `npm run lint`、`npm run build`、`npm run test` 通过；测试为 27 个文件、78 项。
