# 批量删除功能设计文档

## 概述

为历史记录页面添加批量删除功能，允许用户选择多条播报记录并一次性删除。

## 功能需求

### 用户交互流程

1. **进入多选模式**
   - 在历史记录页面顶部添加"多选"按钮
   - 点击后进入批量选择模式
   - 每行记录左侧显示复选框
   - 顶部显示"全选当前页"复选框
   - 顶部右侧显示操作栏：计数器、删除按钮、取消按钮

2. **选择记录**
   - 支持逐条选择（点击复选框）
   - 支持当前页全选（点击"全选当前页"复选框）
   - 支持跨页保持选择（切换页面时保持已选状态）
   - 选中记录背景高亮（浅蓝色）
   - 顶部计数器实时更新"已选 X 项"
   - 删除按钮在未选择时为禁用状态

3. **执行删除**
   - 点击"删除"按钮弹出确认对话框
   - 对话框显示："确定要删除选中的 X 条记录吗？"
   - 如果包含已保存记录，额外提示："其中包含 Y 条已保存记录"
   - 两个按钮："确认删除"（红色）、"取消"

4. **删除后行为**
   - 调用批量删除 API
   - 显示 Toast 提示"已删除 X 条记录"
   - 重新加载当前页数据
   - 如果当前页为空，自动跳转到上一页
   - 退出多选模式

## 技术设计

### 后端 API

#### 新增批量删除接口

**端点**: `POST /api/broadcast/batch-delete`

**请求体**:
```json
{
  "ids": [1, 2, 3, 4, 5]
}
```

**响应**:
```json
{
  "deleted": 5,
  "failed": 0
}
```

**错误响应**:
```json
{
  "error": "请提供要删除的记录 ID 列表"
}
```

**实现逻辑**:
1. 验证 `ids` 参数为非空数组
2. 使用数据库事务批量删除
3. 删除关联的音频文件（broadcast 和 segment）
4. 返回删除结果统计

### 前端实现

#### Store 状态管理

在 `frontend/src/store/index.ts` 中添加：

```typescript
// 新增状态
batchDeleteIds: number[];
isBatchDeleting: boolean;

// 新增 actions
setBatchDeleteIds: (ids: number[]) => void;
batchDeleteBroadcasts: (ids: number[]) => Promise<{ deleted: number; failed: number }>;
```

#### API 客户端

在 `frontend/src/services/api.ts` 中添加：

```typescript
export const broadcastApi = {
  // ... 现有方法
  
  /** 批量删除播报记录 */
  batchDelete: (ids: number[]) => 
    axios.post('/api/broadcast/batch-delete', { ids }),
};
```

#### History 页面组件

修改 `frontend/src/pages/History.tsx`：

**新增状态**:
- `isMultiSelectMode: boolean` - 是否处于多选模式
- `selectedIds: Set<number>` - 已选中的记录 ID 集合（支持跨页保持）

**UI 变更**:
1. 顶部 Header 区域：
   - 正常模式：显示"多选"按钮
   - 多选模式：显示"全选当前页"复选框、计数器、"删除"按钮、"取消"按钮

2. 列表区域：
   - 多选模式下每行左侧显示复选框
   - 选中行背景高亮

3. 确认对话框：
   - 使用模态对话框组件
   - 显示删除数量和已保存记录提示

**交互逻辑**:
- 进入多选模式：清空选择，显示复选框
- 选择记录：更新 `selectedIds` 和计数器
- 全选当前页：选中当前页所有记录
- 切换页面：保持 `selectedIds` 不变
- 点击删除：弹出确认对话框
- 确认删除：调用批量删除 API，重新加载数据，退出多选模式
- 取消：退出多选模式，清空选择

## 数据库影响

无需修改数据库 schema，复用现有的 `broadcasts` 和 `segments` 表。

## 错误处理

1. **网络错误**: 显示错误提示，保持多选模式，保留选择状态
2. **部分删除失败**: 显示成功/失败数量，重新加载数据
3. **空选择**: 删除按钮禁用，不允许提交

## 测试要点

1. 进入/退出多选模式
2. 逐条选择和取消选择
3. 全选当前页功能
4. 跨页选择保持
5. 确认对话框显示（普通记录、已保存记录）
6. 批量删除 API 调用
7. 删除后页面刷新和跳转
8. 边界情况：删除当前页所有记录、删除最后一页记录

## 设计决策

| 决策点 | 选择 | 原因 |
|-------|------|------|
| 布局方案 | 顶部操作栏 | 操作一步到位，视线不用移动 |
| 确认方式 | 二次确认对话框 | 防止误删，用户有撤销机会 |
| 跨页选择 | 支持跨页保持选择 | 适合批量清理场景 |
| 全选功能 | 当前页全选 | 快速选择当前页所有记录 |
| 已保存记录 | 可删除，确认时特别提示 | 给用户明确提示，但不阻止操作 |
| 删除后行为 | 重新加载当前页 | 保持浏览位置，自动处理空页 |
| 技术方案 | 专用批量删除 API | 性能最优，事务保证数据一致性 |
